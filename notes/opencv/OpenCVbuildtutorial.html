OpenCV 编译之旅

最近腦袋有點懵,突發氣象想在自己的電腦上編譯OpenCV.業已證明這是一個據坑.以下是本次旅途所見所地. 系統是Ubuntu 18

 

  • 代码是从两个GitHub仓库下载：1. Opencv 2. opencv-contrib Using git to roll back to the same version ! Github release 页面可以看到不同版本的hash点

 

  1. Do as what the link wrote. https://www.pyimagesearch.com/2016/10/24/ubuntu-16-04-how-to-install-opencv/

    关于是否使用虚拟环境，根据自己的情况。机器性能好，不需要就可以。也可以不使用虚拟环境。

    在cmake opencv时,按照上述连接的命令cmake -D -D -D 会输出相应的配置信息，这个时候要仔细审查配置信息是否正确。 由于ubuntu会自带python2 / python3两个python，因此 cmake 的时候可以制定具体编译的python版本。可以参考下面的制定方式[https://stackoverflow.com/questions/17287250/install-opencv-for-python-multiple-python-versions#]

cmake之后，控制台会报出很多fail，本次实验中遇到的情况均为未安装编译相关的包。遇到的未安装的包的情况大多都在这个链接中列了出来【https://blog.csdn.net/cleanarea/article/details/85846257】

 

 

Another problem Failed to find installed gflags CMake configuration, searching for gflags build directories exported with CMake. -- Failed to find an installed/exported CMake configuration for gflags, will perform search for installed gflags components. -- Failed to find gflags - Could not find gflags include directory, set GFLAGS_INCLUDE_DIR to directory containing gflags/gflags.h [https://blog.csdn.net/qq_15258067/article/details/80033136]

 

另外一个问题是opencv 在ubuntu上面装了两个版本 3.4.1 3.6.1，由于我安装的是3.6.1 ，但是测试的时候 opencv的版本是3.4.1，弄不清楚为什么会默认调用3.4.1，所以我就把电脑上anaconda site-packages & /usr/lib里面相关的3.4相关的东西都删掉。重新编译opencv python2/3 两个版本。

编译安装完成，测试会报错：ImportError: libfontconfig.so.1: undefined symbol: FT_Done_MM_Var [https://github.com/opencv/opencv/issues/12625] 这个问题是因为我之前直接删除了就版本的文件，结果没删除完全!_! [https://github.com/opencv/opencv/issues/12625#issuecomment-437042530]

After deletion, It works fine.
